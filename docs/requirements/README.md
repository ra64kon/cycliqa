# requirements

This folder describes the early requirements, value propositions, and problem frame of the template example.

## Goal

- Make early requirements, target audience, and value hypothesis tangible.
- Capture the starting point for business design and further specification.
- Structure existing example requirements so that roles and the delivery process can continue working with them.

## Getting Started

- `objective.md`: early requirements picture and vision of the initiative
- `use_cases/`: informal requirement sketches and early use case views
- `user_stories/`: verifiable user stories for order intake, preparation, and delivery
