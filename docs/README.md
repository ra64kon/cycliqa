# docs

This folder bundles the documentation entry points of the project.

## Getting Started

- `requirements/`: early requirements, vision, and use cases
- `repository/`: project-wide cross-cutting views on glossary, domain model, components, processes, and data
- `diagrams/`: business and technical diagram sources and visualisations
- `../specs/`: specification hub for business content, software design, and architecture models
- `../specs/business_spec/`: business specification for components, processes, UX, and data
- `../specs/software_spec/`: technical specification for APIs, database, and tech stack

## Structure

- `requirements/`: vision, use cases, and user stories
- `repository/`: cross-cutting and overview views of the project
- `diagrams/`: diagram sources and rendered diagrams
- `../specs/business_spec/`: business result artefacts of the architecture
- `../specs/software_spec/`: technical result artefacts of the architecture

## Reading Paths

- **Read requirements:** `requirements/README.md`
- **Read specification:** `../specs/README.md`
- **Read business specification:** `../specs/business_spec/`
- **Read API documentation:** `../specs/software_spec/api/README.md`

## Note

Requirements are intentionally placed under `docs/requirements/`, while the further specification is bundled under `specs/`.
